## 03-trading-dashboard.html (트레이딩 대시보드)

### 🆕 신규 유저 빈 상태 처리 (Phase 1 구현 시 필수)

**온보딩을 Skip했거나 미완료 유저**가 03 접근 시 환영 카드 표시 필요 (05 참조).

판단 기준:
- `user.onboarding_completed === false` → 환영 카드 표시
- 거래소 미연결 / 전략 0개 / 백테스트 0개 → 단계별 체크리스트

위치: KPI 스트립 위, 페이지 최상단에 dismiss 가능한 카드.

### 인터랙션

| 요소 | 프로토타입 동작 | 실제 구현 |
|------|--------------|---------|
| DEMO/LIVE 토글 | ⚪ 시각만 | 대시보드 전체 모드 전환 + 실거래 경고 |

| "+ 새 봇 만들기" | 🔗 07 전략 생성 | 동일 (또는 기존 전략에서 봇 생성 모달) |

### ⚠️ Kill Switch 특별 주의사항

- 반드시 2단계 확인 다이얼로그 (타이핑 "KILL" 요구)
- 실행 시 즉시 모든 오픈 포지션 market order 청산
- 모든 봇 강제 중지
- 감사 로그 기록 필수
- 30초 쿨다운 (실수 방지)

### 📌 05 각 단계 "건너뛰기" 개선안

현재: "전체 건너뛰기" 만 존재 (All-or-Nothing)

권장: 단계별 건너뛰기
- 각 단계 하단: "이 단계 지금 하기" (primary) + "나중에 하기" (ghost)
- Step 1 (거래소 연결): 필수 경고 + 건너뛸 수 있되 경고 표시
- Step 2~4: 자유롭게 건너뛰기 가능

| 스텝 | 내용 | 실제 구현 |
|-----|------|---------|
| **Step 1: 거래소 연결** | Bybit/Binance/OKX 선택 카드 3개 → API Key 등록 폼 | POST `/api/v1/exchanges/accounts` |
| **Step 2: 첫 전략 업로드** (현재 표시) | Pine Script 붙여넣기 / 파일 업로드 / 템플릿 | POST `/api/v1/strategies` |
| **Step 3: 첫 백테스트** | 자동 추천 설정으로 원클릭 실행 + 결과 미리보기 | POST `/api/v1/backtests` |
| **Step 4: 데모 시작** | 데모 트레이딩 세션 시작 축하 화면 + CTA | POST `/api/v1/trading/sessions` |

## 🎯 공통 패턴 요약

### 1. 액션 버튼 동작 원칙

| 타입 | 프로토타입 | 실제 구현 |
|------|----------|---------|
| **생성 버튼** ("+ 새 X") | 🔗 해당 생성 페이지 | 동일 |
| **편집 링크** ("편집 →") | 🔗 에디터 페이지 | 동일 |
| **삭제 버튼** | ⚪ 대부분 `#` | 확인 다이얼로그 → API DELETE → 토스트 |
| **취소 버튼** | 🔗 목록으로 | `history.back()` 또는 목록으로 |
| **저장 버튼** | ⚪ `#` | API PATCH → 성공 토스트 |
| **실행/제출 버튼** | 🔗 결과 페이지 (성공 시뮬레이션) | 로딩 상태 → API → 성공/실패 분기 |
| **Kill Switch** | ⚪ | **반드시** 2단계 확인 (타이핑) + 감사 로그 |

### 2. 폼 검증

모든 폼은 react-hook-form + Zod v4 사용:
- 제출 전 클라이언트 검증
- 서버 에러는 인풋 아래 빨간 텍스트
- 성공은 토스트 메시지

### 3. 실시간 데이터

WebSocket (Socket.IO) 사용 요소:
- 03 대시보드: 가격 ticker, 포지션 P&L, 봇 상태, 최근 체결, 알림
- 02 리포트 (백테스트 실행 중): 진행률
- 09 목록: 실행 중 행 상태 업데이트
- 08 설정 중: "예상 실행 시간" 재계산 (디바운스)

### 4. 로딩/에러/빈 상태

모든 데이터 fetch 구간:
- **로딩:** Skeleton UI (shadcn `<Skeleton />`)
- **에러:** 빨간 알림 + 재시도 버튼
- **빈 상태:** 일러스트 + 액션 CTA (06에 예시 포함)

### 5. 접근성

모든 페이지 공통:
- 키보드 내비게이션 (Tab, Enter, Space, Arrow)
- Focus visible 링
- aria-live 영역 (실시간 업데이트)
- prefers-reduced-motion 지원
- WCAG AA 대비 확보

### 6. 반응형 (전체 페이지 공통)

- **1440px+**: 최대 너비 컨테이너
- **1200px**: 검색바 숨김 (헤더)
- **1024px**: 사이드바 60px 축소 (아이콘만)
- **768px**: 사이드바 숨김 + 햄버거, 모바일 최적화
