3. Critical / High 결함 상세

BL-340 [P0 Critical] [Confidence H] · Mobile

Trading 페이지 horizontal overflow +227px (375x667 viewport)

재현: 모바일 viewport 에서 `/trading` 페이지가 viewport 602px → horizontal scroll 강제. 375 +227 / 393 +209 / 412 +190 모두 overflow.

기대: 0 overflow.

Root cause: md:grid-cols-4 가 mobile breakpoint 에서도 효과 추정 + 컨테이너 max-w 부재.

Impact: 한 손 사용자가 trading (revenue path) 도달 불가. 가로 스와이프 + 세로 스크롤 = 양손 강제.

BL-339 [P0 Critical] [Confidence H] · Mobile

터치 타겟 19+ 위반 (Apple HIG / Material 44pt 미달)

증거: document.querySelectorAll('button,a').filter(r => r.width>0 && (r.width<44||r.height<44)).length === 19 단 strategies 페이지 한 곳.

Top 위반: UserButton 28x28 / Disclaimer 링크 32-54x14 / 필터 chips 47-83x30.

Impact: 출퇴근 흔들리는 지하철 환경에서 오탑 빈번. 큰 손가락/장갑 사용자 더 심각.

★★★ 공통 발견 · BL-319 + BL-321 + BL-328 [P0, 로컬 dev fix] · Curious + Casual

Clerk dev surface — `accounts.dev` redirect + slug 브랜드 + 영어 form (Embedded 전환 + dashboard 1분 변경 + koKR)

재현: sign-in 클릭 → https://stunning-chipmunk-35.accounts.dev/sign-in redirect. 영어 form. "Sign in to quant-bridge" slug.

Impact: 잠재 고객 결제 의사 50% 손실 (Curious 보고). Casual = 한국어 사이트 첫 단절.

Fix (로컬 dev key 그대로): (1) Clerk Hosted Pages → Embedded `<SignIn /> <SignUp />` 컴포넌트 임베드 (자체 도메인 호스팅 = accounts.dev redirect 제거) (2) Clerk dashboard application name `quant-bridge` → "QuantBridge" 1분 변경 (3) `pnpm add @clerk/localizations` + ``.

분리 — BL-320 "Development mode" 배지: dev instance 정상 표시 (Clerk SaaS feature). 로컬 fix 무의미. Sprint 62 Beta production deploy 시점 → production key (`pk_live_…`) + custom domain DNS 설정 시 자동 해소. 기존 deferred BL-261 영역 묶음.

BL-312 [P1 High] [Confidence H] · QA

OpenAPI / Swagger UI / Redoc 익명 200 — attack surface enumeration

재현: `curl http://localhost:8100/openapi.json` → 200 (97870 bytes 전체 스키마) · `/docs`, `/redoc` 모두 200.

Fix: `FastAPI(docs_url=None if ENV=="production" else "/docs", openapi_url=None if ENV=="production" else ...)` 또는 admin auth gate.

BL-311 [P1 High] [Confidence H] · QA

BE 응답 보안 헤더 0건 (FE 와 비대칭)

재현: `curl -i http://localhost:8100/api/v1/strategies` → X-Frame / HSTS / nosniff / Referrer 모두 부재. `server: uvicorn` info leak.

Fix: `SecureHeadersMiddleware` 추가 + `server` 헤더 strip.

BL-310 [P1 High] [Confidence H] · QA

healthz 503 false-alarm — `_CELERY_TIMEOUT_S=3.0` 너무 짧음 (broker round-trip > 3s)

증거: `inspect(timeout=8.0).ping()` → worker 3개 모두 `{ok:pong}`. 3s 만 부족.

Impact: production K8s readiness probe 시 restart loop 위험.

Fix: env-driven `HEALTHZ_CELERY_TIMEOUT_S` (default 8.0) 또는 `/livez` (broker-skip) 분리.

BL-327 [P1 High] [Confidence H] · Casual

핵심 KPI 라벨에 hover tooltip + 초보 설명 부재

재현: backtest detail → Sharpe / Drawdown / Profit Factor / 승률 라벨 `title` / `aria-describedby` 모두 null.

Impact: 일반인 결과 화면 0% 의미 해독. 도입 결정 차단.

Fix: 각 KPI 라벨 옆 `?` 아이콘 + Radix Tooltip + 한국어 1줄 설명.

BL-341 + BL-342 + BL-343 [P1 High] · Mobile

PWA 0/10 + number `inputmode` 누락 + sheet a11y 시각/ARIA 불일치

manifest / apple-touch-icon / theme-color 부재 → "홈 화면에 추가" 사용 불가.

type="number" 단독 = 소수점 `.` 키 없는 모바일 키보드 → 백테스트 폼 입력 좌절.

BL-322 [P1 Medium] [Confidence H] · Curious

Hero copy vs 실제 기능 GAP — "업로드" 광고 but 직접 입력만

Hero 카피 "TradingView 전략을 업로드하면" but `.pine` 파일 업로드 + TV URL 가져오기 모두 "곧 지원" disabled. 첫 신규 사용자 좌절.
